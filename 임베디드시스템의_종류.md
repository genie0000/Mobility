수행목표: 임베디드 시스템 이해

### 1) 임베디드 시스템 정의
기계나 전자 장비 내부에 특정 기능(목적)을 수행하기 위해 제품 내부에 내장된 마이크로프로세서(컴퓨터) 시스템

### 2) 사용중인 임베디드 시스템
스마트폰, 자동차 ECU, 로봇제어기, 전자레인지, 에어컨, 스마트워치

### 3) 임베디드 시스템을 구성하는 최소 조건
1. 하드웨어: 
    MCU(마이크로컨트롤러) 또는 MPU(마이크로프로세서)
    메모리(RAM, Flash)
    입출력 장치(GPIO, UART, CAN, SPI, I2C 등)
    액추에이터(모터, LED 등)
    센서

2. 소프트웨어:
    펌웨어(Firmware)
    디바이스 드라이버
    RTOS(필요한 경우)
    제어 알고리즘 및 응용 프로그램

4) 펌웨어, RTOS, 임베디드 OS, 각각의 용도
1. 펌웨어
-정의
    하드웨어를 직접 제어하는 가장 기본적인 소프트웨어
    Flash 메모리에 저장되어 전원이 켜지면 바로 실행
-용도
    센서 읽기, LED 제어, 모터 PWM 제어, 통신(UART, SPI, I2C, CAN) 등
    MCU 성능이 크지 않고 동시에 해야 할 일이 많지 않다면 RTOS 없이도 충분
-예시
    전자레인지, 리모컨

2. RTOS
-정의
    실시간성을 보장하는 운영체제
    여러 작업(Task)을 정해진 시간 안에 처리하도록 관리
-용도
    여러 작업을 동시에 수행, 작업 우선순위 관리, 정확한 주기 제어, 실시간 응답 보장
-기능
    Task 관리, Scheduler, Mutex(한번에 하나의 Task만 공유 자원에 접근 가능), Semaphore(허용 가능 갯수만큼 동시 접근 가능), Queue, Timer
-RTOS 종류
    AUTOSAR OS (OSEK를 기반으로 발전), FreeRTOS, ThreadX 등
-예시
    자동차 ECU

3. 임베디드 OS
-정의
    MCU보다 성능이 훨씬 높은 MPU나 SoC를 사용
    임베디드 장치 전체를 관리하는 운영체제
    RTOS보다 기능이 훨씬 많음
-용도
    파일 시스템, 네트워크, USB, GUI, 멀티프로세스, 사용자 프로그램 실행
-예시
    차량 인포테인먼트 등
-대표OS
    Embedded Linux
